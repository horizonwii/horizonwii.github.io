You're talking about the Opening Scene (that plays after you select an empty/new file, in the file select screen in NewerSMBW).

The reason why the Luigi model hasn't changed in the Opening Scene, is because the Opening Scene doesn't use the Luigi Model in the /SpriteTex/Luigi.arc file of NewerSMBW, but it uses it's own Luigi model in the Opening Scene's .arc file (/SpriteTex/OpeningScene.arc) in NewerSMBW.

Posted by newsuperhackboys
Do you guys know what to do fix that?
Edit the Luigi model in the OpeningScene.arc file (the model is located in the g3d/weeg.brres file, inside the OpeningScene.arc file).

Posted by newsuperhackboys
But now, about the message ids. I've edited some messages in NewerSMBW using the Message Editor. The IDs in the editor are different than the ones in Reggie!. I've heard something about Messages.py but I have no idea what that is. Can someone tell me the IDs for the messages or try to help me find them?
In Reggie!, when you reference an ID using the Message Box sprite, you're required to add 256 to the ID (so for example, if the message ID is "10", the ID when referencing it in Reggie!, would be 10+256). That's the reason why the IDs in Reggie! are different from those of the NewerSMBW Message Editor.

If you want to know what ID the Message Box sprite is using in Reggie!, copy the ID into a calculator and subtract it by 256, and that should give you the ID (referenced in the NewerSMBW Message Editor) that the Message Box sprite is using.

(if your region of the NSMBWii disc is "United States / North America" {NTSC-U})
SMNE01.gct to SMNE03.gct

OR

(if your region of the NSMBWii disc is "Europe / Australia" {PAL})
SMNP01.gct to SMNP03.gct

OR

(if your region of the NSMBWii disc is "Japanese" {NTSC-J})
SMNJ01.gct to SMNJ03.gct


(although, I haven't tried renaming the .gct file {for it to work in NewerSMBW} before, so I cannot confirm for sure if it works, but you can try it yourself. If renaming the .gct file doesn't work {and/or the game crashes}, I can't help you there, since I don't know the answer to fixing that problem)

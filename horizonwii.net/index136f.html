Few things :

- You can find .bin files inside tilesets, they are used to store the texture and the animations (both level and map tilesets).
- BMG files don't need to be inside an ARC to be recognized by CTools.
- You can edit BRFNT by using BRFNT2TPL to convert it, editing the TPL and converting back.
- You can find a RSEQ2MIDI on ZementBlock's Google Drive.
- BRTEX can also be opened in BrawlBox
- BTI can be opened in CTools
- You forgot .er files.
- LZ files can be uncompressed by LZ77Mii in Wii.cs Tools.
- You can open .pcm files in Audacity (File>Import Raw Data), but some play at a slower pace than normal.

 The basic idea of tilesets is that you make an image elsewhere, import it, and it becomes a grid of tiles, which you can give properties, and then group them to form objects that are used in Reggie.

A tileset is made by multiple components, and to simplify their descriptions i've compared them to a room designing process.

- Tile : Think of it like a room that has yet to be designed, and is in the early planning stage. It is basically a square of the tileset grid. Tiles can be assigned behaviors, and they make up objects.

- Property : How big will this room be ? Will there be windows ? And where will the doors be ? Properties represent how Mario can interact with the tile. Will it be solid ? Will Mario be able to break it or just pass through ?

- Core Type : Will this room be a bedroom or a kitchen ? Core Types literally define what the tile actually is. Will it be a coin or a pipe, or even a slope ?

- Subtype : Let's suppose we chose this will be a bedroom. If so, which bedroom type ? For kids or for adults ? Subtypes are used to further define the Core Type, such as "Is this a slope ? Then, which part of it this tile represents ?"

- Terrain : This can be compared to the actual room theme. Will it be simple or refined ? Classic or modern ? Terrains literally define what happens when Mario runs over the tile. Will he raise sand or grass stems ?

- Behaviour : It's simply the sum of Property, Core Type, Sub Type and Terrain. No real comparison needed.

- Object : Now that you've designed all the needed rooms, the only thing left is building the house ! Objects are groups of tiles, and they are the ones that actually show up in your level editor, ready to be used.

That's it ! This tutorial was way harder to make, but i hope it will make you easily understand how tiles work

 This tutorial assumes you know what a tileset is and how it works. Here you can find what tilesets are, and here how they work.

Credits go to Orthoptera for this excellent guide.

This tutorial makes use of the Jigsaw Puzzle tileset editor. You can find a download for it in this thread.

Part One : Creating an Image
Images are what tiles actually look like. It's important for them to be good.

You can either create your own image from scratch in Photoshop, Paint.NET or GIMP, or open a different tileset in Jigsaw Puzzle and select "Export Image" from the File menu to edit an existing tileset's image.

The image must be 384x384 pixels, and Jigsaw Puzzle will break it up into 24x24 pixel squares (16 â€¢ 24 = 384). This is important, because your tiles may not be aligned correctly to the grid.

Once you are done making your image, open Jigsaw Puzzle and click "Import Image" from the File menu, and then select your image. This will change the grid of black squares on the left side of the window to a grid of tiles made from your image. Now you can begin adding behaviors.

Part Two : Applying Behaviors to Tiles
Before you start, it's recommended to select "Overlay Collision" above the tiles. This will let you see their shape (square, slope up, etc.).

Now, to make the first tile be the first part of a gradual solid grassy slope, for instance, select "Solid" as a property, the picture of a slope for core type, "Gentle Upslope 1" for subtype, and "Grass" for terrain.

Now you have the behaviors you want on your "paintbrush", and you can "paint" them onto tiles by clicking on them. So click on the first tile and keep your cursor hovered over it, and above the tiles you will see that the first tile has changed to your desired behavior.

In this way you can set behaviors for as many tiles as you need, and you can always check the behaviors for a given tile by hovering over it with your cursor. Before going to the next part, it's a good idea to check all the behaviours you've applied, to make sure you haven't made any mistake.

Part Three : Making Objects
Now switch to the "Objects" tab. Click "Add" to add an object and "Remove" to remove the currently selected object.

You can make the object bigger or smaller vertically or horizontally by clicking the appropriate "+" or "-" button.

You can add tiles to an object just like you can add behaviors to a tile: click on the tile and then click on where you want it to go.

The only other thing to explain here is the menu that reads "Repeat" by default, right next to the "Add" button. It is used to define what happens when you expand the object in Reggie. "Repeat" means just repeat the object, like a brick, the "Slope" options tell Reggie to treat it like a slope, the "Stretch" options tell it to make one part bigger while keeping the rest the same (for instance "Stretch Center" is used on climbable grids to make the middle bigger while still having the same border), and the other "Repeat" options repeat only one part of the object. Most of the time just plain "Repeat" will be fine.

When you're done making all your objects, check that you have applied the correct tiles to them. Then it's time to set the slot and then save.

Part Four : Setting the Slot
All tilesets are set, by default, as Pa0.

Jigsaw Puzzle allows you to change the tileset slot your tileset is meant for by choosing "Set Tileset Slot" from the "Tasks" menu. You should do this for every non-Pa0 tileset you make (because Pa0 is the default). Your tileset's slot is displayed by the expanding-object menu in the "Objects" tab. Once you have set the slot, you can save your tileset.

Part Five : Finishing Up
Choose "Save As" from the "File" menu.

When the naming dialog comes up, you should follow this convention: first the tileset slot, then an underscore, then the name of the tileset, all lowercase, with underscores for spaces, then ".arc"; a name should end up looking like "Pa0_tutorial_tileset.arc".

After that dialog, another one will come up asking you which compression method you want to save with. Choose "Very Slow Compression, Good Quality" option (the other option is faster, but it ruins the image quality a lot).

Jigsaw Puzzle will instantly save the tileset (unlike its predecessor).

Now you can use your tileset in Reggie just like any other one.

 This guide tells you how to run your own mods with Riivolution.

Guide written by FlameKoetsu

Notes :

- IOS37 is required to run Riivolution, the program will ask you if you want to install it (it's safe). If you are on Wii U, see below post.

Steps :

- Download the Riivolution app here
- After downloading the archive, extract it on your SD Card
- Download the New Super Mario Bros. Wii's XML file from here. Right click the Download link and save it.
- Make another folder in the root of your SD card and name it "riivolution" (NO capitals).
- Place the NSMBWii XML into that folder. (Doesn't matter what you name the XML file)
- Now make another folder in the root of your SD card and name it "nsmb" (NO capitals). That is where you'll place the levels you want replaced. You'll have to name the .arc file according to the level you are replacing. Ex. 01-01.arc replaces W1-1. (To find out what the ARC name of levels are, open up the level in Reggie! then the name will be at the top.)
- Run Riivolution how you want (HBC, LetterBomb), enable the NSMB Template Patch and play !
- The "Install" button on the app will make a Riivolution channel on the system menu which allows easy access

Alternative XML files :

- Custom Tileset XML : This should be used if you want to load custom tilesets while playing your mods. Simply follow the included instructions.
- RiiOcarina : This allows you to load games with cheat codes in Riivolution, provided you have the .gct file in the "codes" folder. Just follow the instructions.
- Savegame Redirection : This allows you to save every Riivolution game to the SD Card instead of Wii Memory, so you don't clutter it. Just download it and extract to the SD Card.
- Advanced Testing XML : This is for advanced modders, and contains an organized folder layout, more options in Riivolution and all of the above features, with some extras.

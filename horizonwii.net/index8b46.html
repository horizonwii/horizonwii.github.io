 Since Danster didn't know and 9211tr's guide being confusing, i wrote this one myself.

Requirements :

- The tileset you want to edit
- BrawlBox

Note :

- With this method, you CAN'T change the slot. For that, you need to open the tileset in Puzzle and use the "Set Tileset Slot" option, and save again.

Steps :

- Rename the tileset to whatever name you have (it's advised to follow the Nintendo naming scheme, which is PaX_whateveryouwant (where X is the tileset slot and spaces must be replaced with "_")
- Open the file in BrawlBox
- Go in every subfolder inside the tileset, and rename every file which contains the old name to reflect the filename change (right click > Rename)
- Save the file and you're done !

 After a long while, i finally found out how to edit tileset animations ! Don't get your hopes up though, you can only edit the graphics for each frame (and not the number of frames or the tiles which are animated). Without further ado, let's go !

Notes
- When editing Pa1 images, refer to this picture to know which tiles will be replaced by animated ones in-game. In the higher row the animation frames will be played from top to bottom, while they will be played in the opposite direction in the lower row.

Requirements :
- A Pa0/Pa1 tileset with a working animation (it's recommended to use ones directly from the original game or NewerSMBW, as they are working properly. For this tutorial, i will be using Pa0_jyotyu as a Pa0 example and Pa1_toride as a Pa1 example. You can't add animations to Pa2 tilesets, but you can add them to Pa3 tilesets based off Pa3_waterworks and Pa3_falls (this won't be covered).
- The tileset you want to add animations to
- Jigsaw Puzzle
- BrawlBox (v0.71 is recommended)
- CTools
- A hex editor (i use HxD)

Step 1 : Exporting the Original Animations
- Open Pa0_jyotyu in BrawlBox
- Go inside the BG_tex folder
- Export the three files : block_anime.bin (Brick Blocks), hatena_anime.bin (Question Blocks) and tuka_coin_anime.bin (Outline Coins)
- If you don't need to edit the graphics of any of these tiles, just export the related .bin files and skip Step 2
- Now open each of the exported .bin files with HxD
- Add the following hex data at the start of block_anime.bin (NOT in the column with gibberish text) :
00 20 AF 30 00 00 00 01 00 00 00 0C 00 00 00 14 00 00 00 00 00 80 00 20 00 00 00 05 00 00 00 40 00 00 00 00 00 00 00 00 00 00 00 01 00 00 00 01 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
- Add the following in the same way to hatena_anime.bin ONLY :
00 20 AF 30 00 00 00 01 00 00 00 0C 00 00 00 14 00 00 00 00 02 00 00 20 00 00 00 05 00 00 00 40 00 00 00 00 00 00 00 00 00 00 00 01 00 00 00 01 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
- Add the following in the same way to tuka_coin_anime.bin ONLY :
00 20 AF 30 00 00 00 01 00 00 00 0C 00 00 00 14 00 00 00 00 01 00 00 20 00 00 00 05 00 00 00 40 00 00 00 00 00 00 00 00 00 00 00 01 00 00 00 01 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
- Open Pa1_toride in BrawlBox
- Go inside the BG_tex folder
- Export the three files : belt_conveyor_L.bin (left part), belt_conveyor_M.bin (middle part) and belt_conveyor_R.bin (right part)
- If you don't need to edit the graphics of any of these tiles, just export the related .bin files and skip Step 2
- Now open each of the exported .bin files with HxD
- Add the following hex data at the start of each file (NOT in the column with gibberish text) :
00 20 AF 30 00 00 00 01 00 00 00 0C 00 00 00 14 00 00 00 00 00 80 00 20 00 00 00 05 00 00 00 40 00 00 00 00 00 00 00 00 00 00 00 01 00 00 00 01 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
(this will add the missing TPL header to the file, so BrawlBox can read it correctly)
- Now save all of the files and open them in BrawlBox
- Export the textures and Step 1 is done !

Step 2 : Editing Animations
- Open the newly exported textures in your image editor of choice (just don't resize the texture)
- Each tile is 32x32, so keep that in mind when editing
- When you're done, save the textures
- Open the edited .bin files in BrawlBox and replace the textures, then save and close the program
- Open the .bin files in the Hex Editor, then remove the extra data you added in Step 1 in each file.

Step 3 : Making a New Tileset
- Open Jigsaw Puzzle (if you only have a tileset texture, skip the next point)
- Open your tileset and export the image (File>Export Image)
- Now open Pa0_jyotyu/Pa1_toride (still in Jigsaw Puzzle)
- Import the tileset image, then remake the tileset as if you were doing it from scratch (see here if you need help on this)

Step 4 : Finalising
- Open the saved tileset in CTools' SZS Explorer
- Now replace all the .bin files in and BG_tex with the ones you created at the end of Step 2 by right clicking and selecting Replace (or click Import>File if they aren't already present)
- Save, and you're done !

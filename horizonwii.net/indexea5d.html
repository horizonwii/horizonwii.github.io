Also if you're using custom tilesets in NSMBW with Riivolution, you need to replace the line in your XML:
<Wiidisc version="1">
with
<wiidisc version="1" shiftfiles="true">
or the game will crash, regardless whether the tileset exists or not.

With Dolphin emulator, I believe you can only have custom tilesets, only if you replace an existing tileset file in the ISO, it cannot be a custom filename or the game might crash.

Also one more tip I forgot to add, in case if people are having trouble with the blue warp pipe sprite (it crashes when entering it in the game):

If you're using the blue warp pipe (I believe it's the Enterable Warp Pipe Facing Up) sprite, the entrance you put on the pipe, must match the entrance ID of the blue warp pipe sprite, so for example, on the blue warp pipe, if you put an entrance ID of 9 on it, you must change the entrance ID of the blue warp pipe sprite to entrance ID 9, for it to work.

And I don't know if this is required or not, but you might also have to set the entrance type (of the entrance on the blue warp pipe sprite) to the "Normal" entrance type.

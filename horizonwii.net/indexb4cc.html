 Soo, after Danster's achievement here i am with a ready-made guide, which will allow you to customize the original NSMBW World Maps. Credits go to Danster for testing and John10v10 for providing valuable help, i just had the idea...

Please note that most of the content of these maps is hardcoded, and you cannot remove it without getting the game to crash. However, there are several modifications you're allowed to make.

NOTE : Save a lot, just save whenever possible, because if your map crashes you'll have to restart from scratch.

Textures - Easy

Requirements :

- BrawlBox
- CS_WX.arc (found in /WorldMap, it's the map model)
- WS_WX.arc (optional, it's the mini-model from the map select screen, which you can change to match the new world theme)
- Env_world.arc (optional, it contains the lighting effects, in case you want to further change the theme)
- Post-Effect Editor (optional, for editing the effects)
- Color Codes (optional, to choose the color)

Steps :

1) Open CS_WX.arc in BrawlBox
2) Move to g3d > model.brres > Textures and edit any texture you want, then save
3) Done ! That was easy, eh ?

Editing Liquid Colors :

See this guide.

Editing the World-Select Map :

1) Open WS_WX.arc in BrawlBox
2) Move to g3d > model.brres > Textures and edit any texture you want, then save
3) And this one wasn't too hard, too.

Editing the Lighting Effects (Advanced) :

Pro-tip : You can copy color data from another world's lighting, if you don't want to make them yourself

1) Open Env_world.arc with BrawlBox
2) Move to light > WX.blight and export the file (you don't need to edit the BLMAP as it controls the lighting placement and its format is mostly unknown)
3) Open Post-Effect Editor
4) Click File > Open and open the .blight file
5) A table will appear with lighting data of the map, now you can edit the entries as explained in the next step
6) The only rows you need to edit are Red, Green and Blue, both in Light Objects and Ambient Lights (you can safely ignore the entries which have all three set to FF, don't touch anything else)
7) Use the color codes site to check what colors the lights represent (write the values in the hex field, marked with "#")
8) After replacing all the data you need, save the file
9) Reimport the edited .blight file using BrawlBox, then save

Level Node Editing - Medium

Note : You can't move or add nodes, only change what level the node loads and what it's represented by (Tower/Castle/Toad House...)
Note 2 : Adding a normal stage where previously there wasn't will not make this stage appear in the Star Coin screen or in the Free-for-All and Coin Battle selection screens. However, it will count towards the Star Coins and the completed exits totals that award the player with stars.

Requirements :

- BrawlBox
- Notepad++ (for the love of god, don't use Microsoft Excel or the files will get corrupted)
- CS_WX.arc
- RouteInfo.arc (same folder as the file above)

Steps :

1) Open CS_WX.arc in BrawlBox
2) Move to g3d > model.brres > 3DModels

Now you have to find the level node you want to edit, open the spoiler below to see what they correspond to.

Note : Y is replaced by the world number, while Z is explained nearby

- WY01 : Level -1
- WY02 : Level -2
- WY03 : Level -3
- WY04 : Level -4
- WY05 : Level -5
- WY06 : Level -6
- WY07 : Level -7
- WY08 : Level -8
- WY09 : Level -9 (unused)
- WYG0 : Ghost House
- WYT0 : Tower
- WYT1 : Tower 2 (unused)
- WYC0 : Castle
- WYC1 : Castle 2 (unused)
- WYA0 : Ship
- WYKZ : Toad House (lower numbers are nearer to the world start, they go in order)
- WYW0 : Cannon
- WYSZ : World Start (the node with the arrow, use S0 for an horizontal one and S1 for a vertical one)
- WYX0 : Peach Castle

3) Now go in CS_WX (W3 has CS_W3a and CS_W3b, so you may have to edit both)
4) Go in Bones > CS_WX and replace every instance of the old name with the new one (NOT only in the ones listed in the spoiler, but in every bone which contains it as a part of its name)
5) To edit the level representation, open the bone and check below :

- To remove the representation, delete the child bone
- To edit the representation, simply rename the bone (check the list in the second spoiler)
- To add a new representation, open the spoiler (it's a bit long) :

1) Right-click on any child bone from any level node, and select Extract
2) Right-click the node you want to add the representation to and choose Add New Child
3) Replace the new bone with the exported one by right-clicking and pressing Replace (don't worry if the BoneIndex value is wrong, BrawlBox will fix it by itself)
4) Rename the bone using the next spoiler

- cobKinokoRed1/3 : Red Toad House (number 3 is only used when there's multiple Red Toad Houses)
- cobKinoko1up0 : Green Toad House
- cobKinokoStar2/3 : Yellow Toad House (number 3 is only used when there's multiple Yellow Toad Houses)
- cobGhost : Ghost House
- cobToride : Tower
- cobCastle : Castle
- cobKoopaShip90 : Airship
- cobKoopaCastle : Final Castle
- cobCannon/00/1 : Cannon (i don't know the difference between the numbers, just follow the name of the cannon in the world you're editing)
- cobPeachCastle : Peach Castle

6) Move to Vertices
7) Now replace any instance of the old node name with the new one
8) Do the same for Normals and Colors
9) Move to Objects
10) Check every polygon's VisibilityBone parameter until you find the old node name, then replace with the new name
11) If you have deleted a representation, delete the related polygon (should be near the node)
12) Move to AnmChr > CS_WX
13) Now replace any instance of the old node name with the new one
14) Do the same in AnmChr > CS_WXsound
15) Save the file, and open RouteInfo.arc
16) Go in the folder of the world you want to edit, and export the .csv files in there
17) Open the exported file with Notepad++
18) Now replace any instance of the old node name with the new one
19) Save the file and switch to routeWX.csv
20) Repeat step 18 for this file, then save
21) Reimport the two files in BrawlBox, save, and you're done !

Level Exit Editing - Medium-Hard :

Requirements :

- BrawlBox
- Notepad++ (for the love of god, don't use Microsoft Excel or the files will get corrupted)
- RouteInfo.arc

Steps :

1) Open RouteInfo.arc in BrawlBox
2) Go in the folder of the world you want to edit, and export the file pointWX.csv
3) Open the file with Notepad++
4) Find the stage of which you want to edit the exits (refer to the first spoiler in the previous paragraph)
5) The routes inside the second set of quotes will be unlocked with a normal exit, while the ones inside the third quote will be unlocked with the secret one, the first set's usage is unknown
6) Use the parameters already inside the quotes to make sure not to do it wrong
7) After editing, save the file and you're done !

Editing Route Animations - Hard :

Note : for route animations i mean the animations that the character uses while on a specific part of the map.

Requirements :

- BrawlBox
- Notepad++ (for the love of god, don't use Microsoft Excel or the files will get corrupted)
- RouteInfo.arc

Steps :
1) Open RouteInfo.arc in BrawlBox
2) Go in the folder of the world you want to edit, and export the file routeWX.csv
3) Open the file with Notepad++
4) Now this will be harder than normal editing, as the routes are obscure and the animations have Japanese names, so follow this closely
5) If the characters are gibberish, click Encoding > Character Set > Japanese > Shift-JIS
6) At the start of the row are the points that are connected by the route (most aren't level nodes, but it will be easy to find where they are as they are oredered starting from the world start node
7) After the comma will be the character animation, you can edit them by using the following values (most of them are unsure, we need people that know Japanese to confirm these) :

- 道 (translates to Road) : the character will walk to the next route without any particular effect
- ジャンプ (translates to Jump) : the character will jump to the next route
- 木 (translates to Wood) : the character will walk as if it was on a wooden surface
- はしご右 (translates to Ladder Right) : the character will climb a ladder that is on the right
- はしご左 (translates to Ladder Left) : the character will climb a ladder that is on the right
- はしご岩 (translates to Ladder Rock) : the character will climb a ladder made of rock
- ツタ (translates to Ivy) : the character will climb a vine
- 氷坂 (translates to Ice Slope) : the character will slide on ice
- 氷 (translates to Ice) : the character will walk with a slippery effect
- 雪 (translates to Snow) : the character will emit snow puffs when walking
- 土 (translates to Earth) : possibly dirty terrain ? (from W5)
- はしご (translates to Ladder) : another ladder ? (from W5)
- ﾍｻ (translates to Hesa) : possibly sand ? (from W2)
- ﾂ催｢ (translates to Urge) : possibly runs faster ? (from W3)
- 窶｢X (translates to Rustic) : i don't know what this could mean (from W3)
- ﾓｹ (translates to Moke) : uhh (from W3)
- ﾃ療ε禿ｶ (translates to Therapy Baldness) : uhhhhhh (from W2)
- ﾂ債ｵﾂｲﾓｪ (translates to Tsubuki Otsuyoe) : what the... (from W3)
- ﾂ静｡ (translates to Tsu Static) : whatever (from W3)
- ﾗｬﾍｻ (translates to Rajaega) : oof (from W2)
- 窶｢Xﾂ催｢ : doesn't even translate (from W2)

And that's it for now, further guides will be added if more things are found.

 The source version of Reggie Next adds many more improvements to the latest compiled version, so here's a guide on how to run it. The pictures only apply to the Windows version.

Prerequisites
- Reggie! Next M3 Source
- Python 3.4.4
- PyQt 5.4.1
- TDM-GCC (Windows ONLY, skip the related steps if you're on OS X/Linux)
- Cython
- NSMBW's or NewerSMBW's Stage Folders

Installing Python
Go to the directory you downloaded the installer to and open it. You will be welcomed with this :



Now press Next and again Next, and you will see this screen :



Click on the disk (or cross) icons to match the settings in the picture above, then click on Next and let the installation complete itself, then press Finish to close the installer. To test if Python was installed correctly, press WIN+R, type cmd and press OK.



After that, type "python" in the terminal window. This is what you should get (exact text may vary between systems) :


Installing PyQt
Go to the directory you downloaded the installer to and open it. You will be welcomed with this :



Press Next and then I Agree. After that, you will see this screen :



Set the checkboxes as in the image above, then click on Next and again Next, and let the installation complete itself, then press Finish to close the installer. To test if PyQt was installed correctly, open cmd (you know how to do it now), and once it opens, first type "python" and then "import PyQt5". If nothing happens, it's installed correctly.


Installing TDM-GCC
If you are on OS X/Linux, skip all the steps inside this spoiler and go to the next one.

Go to the directory you downloaded the installer to and open it. You will be welcomed with this :



Press Create to make a new installation, then press Next and again Next. You will see this screen :



Edit the installation path to be the same as in the picture above, then press Next and again Next. You will now be shown this screen :



Toggle the options as shown in the picture above, then click Next and let the installation complete. On the window that appears, toggle the checkbox and press Finish.



To test if GCC was installed correctly, open cmd (you know how to do it now), and once it opens, type "gcc --version". You should get something similar to this :


Installing Cython
If you are on OS X/Linux, skip to the last step inside this spoiler.


Go to the folder C:/Python34/Lib/distutils and create a file named "distutils.cfg".



Open the newly created file in Notepad, place inside the following and save it :
[build] compiler = mingw32



Now open cmd and type "pip install Cython==0.25.2"



Opening Reggie! Next
Double click reggie.py to open the program. It will ask you for the Stage folder from NSMBW, of which you should have a copy if you have followed the tutorial linked earlier. Open the dump folder, and choose the folder named "Stage".

Importing NewerSMBW's Stage Folder
Extract the .7z file you've downloaded earlier (only the folder "Stages" and "Tilesets" are required)



Now move the "Tilesets" folder inside the "Stages" folder, then rename it to "Texture". This should be the result :



Open Reggie! Next, click on File > Change Game > Newer Super Mario Bros. Wii, then point the program to the "Stages" folder you've just prepared.


Troubleshooting
Did you get the "failed to execute script reggie" error ? It can be fixed by resetting the program through the registry editor. To open it, press WIN+R, type "regedit" and click OK.



Once in, browse to HKEY_CURRENT_USER > Software > Reggie, right click on "Milestone 3 Alpha 2" and press Delete.



Now you can reopen Reggie! Next, which should work properly now.

 Great tutorial!!
There's one paragraph from the KB with additional info:

If you have installed Python 2.7 after 3.4.x, or you did not associate .py files with Python 3.4.x, then Reggie! might not start up, because .py files are still associated with Python 2.7. In this case, you have to use the following command to run Reggie!:
"c:\python34\python.exe reggie.py"
Run this command from the folder where Reggie! was extracted.
You can also create a .bat file to automate the process.

If libEGL.dll is missing, you didn't copy it to the Reggie! Next folder:
Posted by customlevelfan78

If copying the libEGL.dll file doesn't work, then try to run Reggie! Next, using the included .bat file (in the Reggie! Next folder).

If it still doesn't work (and you've tried running Reggie! Next with the .bat file), and you're running the 64-bit version of Python/PyQt/cx_Freeze/etc., try uninstalling Python/PyQt/cx_Freeze/etc., and try installing the 32-bit (x86/x32/Win32) version of Python/PyQt/cx_Freeze/etc., and try re-compiling Reggie! Next again.

 With this tutorial you can edit colors of liquids in NSMBW Maps (such as World 1) and Sprites (such as Water, Lava...). Please note that this is what i remember about it, so some details might be wrong. If you find any, i would be happy if you reported them.

Requirements :

- The map or object you want to edit
- BrawlBox
- HTML Color Codes

Steps :

- Open the file in BrawlBox
- Go into 3DModels > MDL0 > Material (if a sprite is made by multiple models, do the following for all of them)
- Find the material used by the liquid and select it
- Look for TevColorBlock in the Material
- Go to the linked website and convert the color you wish to use
- Now edit the color codes (DO NOT touch the alpha value)
- Save the file and you're done !

 This is a tutorial on how to put your own music in NSMBW.

Requirements :

- The song you want to edit
- NSMBW's BRSAR file (wii_mj2d_sound.brsar)
- An audio editor (i recommend Audacity)
- BrawlBox
- Better BRSAR Patcher
- BRSAR Patcher (for the old method ONLY)

Steps

Pre-Step :
1) Check this thread. It contains a lot of useful information, such as filenames, channels, required loops...

Making the BRSTM :
1) Open the song in Audacity.
2) Listen to the song carefully, so you may get an idea of where the music should loop (skip this if you don't want/need a loop)
3) Click on File > Export as WAV
4) Open BrawlBox and press File > New > Brstm Audio Stream.
5) It will ask you to open a .wav file.
6) BrawlBox gives you several options, such as the loop.
7) The loop box is already ticked (untick it if you don't want/need a loop)
8) Move the two bars and/or edit the sample numbers to set your loop, and listen to it to see if it sounds good (skip this if you don't want/need a loop)
9) If the music you're replacing is multi-channel, write down the loop numbers you've set and go on by following this thread (skip step 6 of that tutorial as you've already done what it says)
10) Press Okay, BrawlBox will create your BRSTM file.
11) You now need to save the file, refer to this thread to choose the correct filename.

Further notes :
1) If you need to create a speeded-up version of your music, check this thread.
2) If you're replacing one of Newer's .er files, simply rename the file extension from .brstm to .er.
3) Make sure to not loop songs that don't need it ! For example, if you loop the "death" sound, you will keep hearing it over and over again together with other songs.

Patching the BRSAR :
This method is much faster and supports bigger BRSTMs. Note : If you're replacing any of Newer's .er files, BRSAR patching is not needed, so you can skip this part.

1) Put all your BRSTM files and the BRSAR (MAKE SURE you've got the filenames right) in a folder together with Better BRSAR Patcher.
2) Drag the BRSAR inside the batch script, then let it do its job.

Patching the BRSAR (OLD METHOD) :
Note : If you're replacing any of Newer's .er files, BRSAR patching is not needed, so you can skip this part. This method doesn't support BRSTMs bigger than 16MB.

1) Open the BRSAR Patcher
2) Find the .brsar file
3) Select the song you want to update in the Music Track option
4) Click on the button with the music note and open your BRSTM file.
5) Press on the save button to patch.
6) Repeat for every changed song, and make sure to not forget the faster versions !

That's it !

 This tutorial teaches you how to create a Map Tileset from scratch, starting from a normal level tileset. Credits for the template go to Tempus.

Requirements :

- The tileset you want to convert for use in maps
- Puzzle
- Photoshop (any version above and including CS3 should work)
- Koopuzzle Tileset Template
- Koopuzzle
- BrawlBox

Pre-steps :
1) Open Photoshop
2) Double click NSMB Worldmap Template.atn
3) Open NSMB Worldmap Template.psd

Making the tileset image :
1) Open the tileset you want to convert with Puzzle
2) Click on File > Export Image and save it
3) Open the exported image in Photoshop
4) Take six 'top terrain' tiles, and copy them to the spaces with the blue stars
5) Take twelve 'inner terrain' tiles, and copy them to the yellow starred area
6) Take six 'bottom terrain' tiles, and copy them to the green starred area
7) Create a nice sizeable pattern that tiles to 24x24 for the ground (Keep it bright and colourful and use a geometrically repeating pattern for best effect)
8) Select that pattern (in a box a multiple of 24x24) and go Edit->Define Pattern...
9) With your new pattern, select the Layer 'Put your Pattern Here', and use Edit->Fill->Pattern Fill
10) Run the included Photoshop Action Script, "Make All" (make sure the Folder 'Terrain Bottoms' is open, or it will eventually error)
11) Add deco tiles as desired to any of the red starred spaces
12) Export to PNG

Creating the actual tileset :
1) Open Koopuzzle
2) Click on File > New Tileset from Image and import the png you just made
3) Now it's time to make the objects, check the spoiler if you don't know how to do that (it works similarly to Puzzle).
Click "Add" to add an object and "Remove" to remove the currently selected object.

You can make the object bigger or smaller vertically or horizontally by clicking the appropriate "+" or "-" button.

You can add tiles to an object easily : click on the tile and then click on where you want it to go.

The only other thing to explain here is the menu that reads "Repeat" by default, right next to the "Add" button. It is used to define what happens when you expand the object in Koopatlas. "Repeat" means just repeat the object, like a brick, the "Slope" options tell Koopatlas to treat it like a slope, the "Stretch" options tell it to make one part bigger while keeping the rest the same (for instance "Stretch Center" is used on climbable grids to make the middle bigger while still having the same border), and the other "Repeat" options repeat only one part of the object. Most of the time just plain "Repeat" will be fine.

When you're done making all your objects, check that you have applied the correct tiles to them. Now you can save.

4) Click on File > Save, and give it the name you want. The program will generate an .arc file, which you can use in Koopatlas.

Making your tileset Newer-compatible :

The tileset you just saved is in .arc format, it can be used in Koopatlas, but it CAN'T be used in Newer. To make it compatible follow these steps :

1) Open the .arc file in BrawlBox
2) Look for a file called "optimisedTexture.bin"
3) Export that file and give it the same name as the .arc file (WITHOUT the file extension)
4) The new .bin file can now be moved to the Maps/Texture folder.

As i stated, DO NOT delete the .arc file, or you won't be able to use your tileset in Koopatlas !

That's it, you're done !

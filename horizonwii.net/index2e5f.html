 Repost from StapleButter's thread on Kuribo64 with additions by me and ZementBlock. Happy read !

Posted by StapleButter
Requirements :

- All of NSMBW's files extracted in a folder (see here to know how to do it, you MUST use method number 3)

Steps :

1) Look for the file "apploader.img" in the dump you've made (it MUST be there, or you can't go further)
2) Open Dolphin
3) Go to Config
4) Go to the Paths tab
5) Set the DVD root path to the folder containing the game files and set the apploader path to apploader.img (if it isn't there, you NEED it)

Now, to make Dolphin see your game's save data, there's an extra step needed. You can skip it altogether if you don't have any :

1) Open Dolphin's "User" folder (it's the NAND folder in the Paths settings)
2) Go into title > 00010000, then in one of these folders (if you have them) :
534d4e50 - NSMBW PAL
534d4e45 - NSMBW NTSC-U
534d4e4a - NSMBW NTSC-J
534d4e4b - NSMBW NTSC-K
534d4e43 - NSMBW NTSC-C

3) Copy one of the folders you have found
4) Go back to the "title" folder
5) Create a folder named "ffffffff" (if it already exists, rename it or delete it).
6) Go inside that folder, paste the folder you previously copied and rename it to "ffffffff".

You can then launch Dolphin, and load the game's main.dol, which will start the game as if it was running from an ISO/WBFS. That's it !

Note : To apply file changes while using this method, simply place the files where due while the game isn't using them. Dolphin will crash if the new file is bigger than the original, and a restart will fix the issue.

Open the NSMBW ISO you extracted with WIIScrubber and open the DATA partition of the ISO (which is normally "Partition 1"), and right-click and click Extract..., in WIIScrubber, "one-by-one", on the needed files (the apploader.img, fst.bin, and boot.bin).

And the sys folder is a folder that you must create (to put the apploader.img, fst.bin, and boot.bin files in), it isn't available in the extracted NSMBW ISO or such.

The .LH file is just a compression to the .arc file, mainly to reduce space in the NewerSMBW mod.
So it isn't required, you can simply use the normal 01-01.arc (or whatever else file) in NewerSMBW, uncompressed, without the .LH compression.

Just don't rename the normal/uncompressed file (e.g. "01-01.arc") to put the .LH file extension in it (e.g. "01-01.arc.LH"), or the game will think it's compressed and will try to decompress it (and it'll fail/crash, since the file is uncompressed).

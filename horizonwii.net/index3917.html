 So, your level crashed the game or got Dolphin stuck ? There are a lot of possible causes, and this thread tries to cover them all.

NOTE : You should always test on real Wii and NOT on Dolphin, as it can handle things that the Wii can't. Make sure your content works on Wii before releasing it.

First steps (follow these BEFORE checking for anything else) :
These steps will make use of the Level Diagnostic Tool to make sure everything is right. You'll need Reggie Next Public Beta 1 (which you can find here) or Milestone 3 (here). Milestone 2 Alpha 3 reports nonexistent errors so it's not recommended.

1) Open your level in Reggie Next
2) Click on "Edit", then on "Level Diagnostic Tool"

The possible causes are :

Some objects are not found in the tileset files : This means your level contains objects which your tilesets doesn't include, and will crash the game upon load. This usually happens after switching a level's tileset. On the latest M3 release you can find these tilesets, as they are marked by a blue-ish square with a red question mark in it.

There is no Start Entrance in this area : This only matters in Area 1, and it means that the Start Entrance you pointed the game to doesn't exist.

There are too many sprites in this area : You've went too far with the sprites, remove some. According to ZementBlock, this limit is 482 sprites per area, but this is untested.

Two zones are too close together : Having two or more zones overlapping or near each other causes the game to behave weirdly, and most times it will cause a crash. For the reference, zones must be placed at a distance of at least 10 blocks from each other.

These are all the errors that can be detected by the diagnostic tool. The next ones will have to be found by yourself. You may find other errors, but they do not matter if they're not marked in red.

Extra Content-related errors :
Buggy Extra Content : Make sure the custom content that the level makes use of works fine. This includes : tilesets, backgrounds, sprite retextures, characters, HUD, font, code edits. For first tests use vanilla NSMBW or NewerSMBW with only the basic content that the level requires.

Incomplete XML : Make also sure these extra mods you're using are loaded by the XML, or they won't be applied and the game will not find them.

Cheat Codes : Disable any cheat code or hook before testing, or they may lead to crashes.

Area-related errors :
Number of Zones : Using more than 7 zones in a single area will cause the game to crash, Reggie will alert you, but you may have ignored it.

Zone-Related errors :
Size : There is an hardcoded limit to the size of a zone, and passing it will lead to a crash. The limit is of 10000x1800 for an horizontal (and 1800x10000 for a vertical one), while the limit for a square zone is of 3600x3600 (this one value depends on the zone position and some other factors, which i currently don't know). This also applies to the sum of the zones in the area, to measure this simply find out if all the zones fit in the rectangle/square.

Background : You may have set a background that does not exist. Make sure you've set it right.

Music/Echo : You may have set a soundtrack ID/echo that does not exist. Rare error, though.

Sprite-Related errors :
Forgotten IDs : Generally, never point a sprite to something that does not exist, such as a location, a path, an entrance or a controller (also check that the pointed entrance is enterable), or anything that the sprite requires to work. Setting such value to 0 won't harm in most cases.

Multiple IDs : While you should never point sprites to non-existant things, some elements should NEVER have the same ID, including Locations, Paths, Entrances, Rotation Controllers, Movement Controllers and more. Having the same Event ID though can be intentional and work fine.

Buggy Settings/Sprites : If the sprite tips tell you that something may not work, you better take the advice, or you'll risk getting a crash. The same thing applies to requirements, if a sprite requires another to work, don't ignore that tip.

Entrance-related errors :
Unenterable Entrances : Having an unenterable starting entrance, or sprites (such as Special Exit Controller or anything with pipes) may lead to a crash. Check all your entrances before testing the level.

Non-existant Entrances/Areas : Having an entrance that points to a non-existant entrance or Area will cause a crash as soon as you access the entrance. As i said, check them before testing.

Connected Pipes : If you're using connected pipes, make sure to point the entrances to the correct path.

After checking all these, you should be fine. If you're still having issues, feel free to make a post in the NSMBW Questions thread and we'll try to resolve your problems

Also if you're using custom tilesets in NSMBW with Riivolution, you need to replace the line in your XML:
<Wiidisc version="1">
with
<wiidisc version="1" shiftfiles="true">
or the game will crash, regardless whether the tileset exists or not.

With Dolphin emulator, I believe you can only have custom tilesets, only if you replace an existing tileset file in the ISO, it cannot be a custom filename or the game might crash.

Also one more tip I forgot to add, in case if people are having trouble with the blue warp pipe sprite (it crashes when entering it in the game):

If you're using the blue warp pipe (I believe it's the Enterable Warp Pipe Facing Up) sprite, the entrance you put on the pipe, must match the entrance ID of the blue warp pipe sprite, so for example, on the blue warp pipe, if you put an entrance ID of 9 on it, you must change the entrance ID of the blue warp pipe sprite to entrance ID 9, for it to work.

And I don't know if this is required or not, but you might also have to set the entrance type (of the entrance on the blue warp pipe sprite) to the "Normal" entrance type.

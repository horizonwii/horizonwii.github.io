 Welcome to the Sprite Usage Tutorial ! In this post i listed every sprite and how to use it correctly in your levels. Search the sprites you want, open the spoilers and have a good read ! (This is not done yet, and will take a lot of time to complete, if ever)

Sprites and/or settings marked with an asterisk are only available in NewerSMBW.

Remember, this is not meant to be strictly followed, it just provides guidelines to use them in levels correctly.

NO ASSIGNED FUNCTION (Sprite 10-19, 22,...)
These sprites do nothing, they are completely useless. There's no need to place them in your level.

Character (Sprite 0-9)
Sprite Image available : Reggie Next only
Yoshi Compatibility : Unknown
Notes : Sprite 9 is the most stable one

Description :
This sprite creates a stationary clone that (in normal Reggie) will be shown one tile above the sprite's placement. The clone will not move; nonetheless, the 'twinkle' animation works. Also, this clone can shoot fireballs if the player has a Fire Flower. It seems the initial purpose of this sprite was to function as an entrance. Additionally, depending on the position of this sprite relative to an entrance, this clone can sometimes be controlled instead of the one created by the entrance.

Recommended Usage :
None. This sprite is really buggy, and is pretty much useless for everything, because it does not move.

Settings :
- Looks Left : No need to explain
- Entrance Type : Same list as the normal entrances
- Character : You can choose one of the four playable characters. Please note that this requires the actual player to work, or the game will freeze (e.g spawning Blue Toad in Single-Player)

*Koopa Throw (Sprite 10)
Sprite Image available : No
Yoshi Compatibility : No

Description :
This is used in the final Bowser fight. It causes Bowser to throw objects at Mario, if he reaches a predefined position.

Recommended Usage :
While the functions of this sprite are included in Sprite 213 (Captain Bowser), this allows more customization (Z-Order, thrown object, throw direction...)

Settings :
- Is In Front : With this enabled, thrown objects will pass over other on-screen elements and not behind them
- Type : You can change the thrown object to any of these
- Throws Right : Explains itself.

*Roll Your Own Model Sprite (Sprite 11)
Sprite Image available : No
Yoshi Compatibility : Yes

Description :
This sprite can load custom models from the SD Card, which you can inject yourself, or just use the available ones. The 3D Models can be easily swapped, but have to reside in the same file, otherwise the game will crash. Just don't swap 3D Models or Animations between different files.

Recommended Usage :
- Most models are used in cutscenes or title screens, but can be used to make some great scenery.
- The sprite containing the model needs to be placed in the level as well, or the game will crash. In the spoiler below is a list of models conveniently divided in categories, with description and the required sprites.

These 3 models are used in different places and have different requirements (underlined)

- Cloud : A cloud, used in World C-2, needs Loose Arrow
- Tree Canopy A/B/C : Autumn tree foliage of different colors, they need Chestnut Goomba
- Tree Canopy D : Green tree foliage, used in 1-Tower, needs Chestnut Goomba

- Opening Balloon Green : Green Balloon

- Opening Toad A : Yellow Toad
- Opening Toad B : Blue Toad

- Mario in a Clown Car : Mario

These 2 models are unused, but i don't know how they look and what their requirements are.

- Peach Castle Backdrop : Possibly a Peach Castle you can put in your background in any level ? This is just speculation.
- Backdrop : I can't even speculate for this one. The only thing i know is that this shouldn't be used in levels, because it may cause crashes.

Settings :
- Scale : Changes the Scale of the 3D Model. Value 0 is 100%, while other ones progress by 40 (1 is 40%, 2 is 80% and so on...)
- Is Animating : Some models have animations available for them, which changes both their look and their behaviour. Enabling animations for models that do not support them or don't have them will crash the game. Some models have been confirmed to work, but documentation for this is near to none.
- Z-Order : Changes the distance of the model from the player. Negative values are behind Layer 1, and positive ones are above it. Some in-game values :

- Background: -5500
- Layer 2: -3300
- Layer 1: 0
- Player: 300
- Foreground: 1200
- Layer 0: 3300
- Above Layer 0: 5500.0

*Star Collectable (Sprite 12)
Sprite Image available : Reggie Next only
Yoshi Compatibility : Unknown

Description :
The mini stars used in the Challenge Houses in Newer. When all the stars are collected, an event is triggered.

Recommended Usage :
- In an entire hack, unless you are planning to not use Challenge Houses in it, keep this element for those houses only.
- If you are making a single level though, they would be a pretty good puzzle element for your stage, requiring the player to get all the stars before being able to progress/unlock things in the level. Obviously don't go crazy with these, or the game will lag.

Settings :
- Triggering Event ID : When all the stars in the zone with this setting on the same value are collected, this Event ID is triggered. The triggered element can be anything : Chainers, Actor Spawner, Red Coins...
- Spawn if Player X Exists : This setting is extremely useful to tune the difficulty based on the number of players. Please note that the first value is 0 (always shown), and not 1 (only if 2nd player is playing) !

*Rideable Clown Car with Cannon (Sprite 13)
Sprite Image available : Reggie Next only
Yoshi Compatibility : No

Description :
The Clown Car used in the Newer Boss fight. The player can fly around freely and shoot Cannons.

Recommended Usage :
- This sprite has very few interactions with other elements, so it should be used only in special occasions (such as a boss fight). Inside the spoiler is a few examples of interactions and bugs caused by them.

- Captain Bowser : Can be shot at
- Asteroid : Can be shot and destroyed
- Blocks that can be destroyed by Bob-Ombs : Can be shot and destroyed (this may not end well if it happens in a normal level)
- Most enemies : Can't be shot at, but the player can still stomp them. This will cause a bug : the car will leave the player, but he can still control it from far away. He can re-enter it by ground pound, but this bug is pretty bad.

- Due to the above, use this sprite in zones with enemies that Mario can't kill by stomping.

Settings :
None.

*Koopa Flame (Sprite 14)
Sprite Image available : No
Yoshi Compatibility : No
Note : Requires Captain Bowser (Sprite 213)

Description :
Causes Bowser to shoot flames when Mario approaches the sprite.

Recommended Usage :
Since it requires Captain Bowser, this can only be used during the boss fight.

Settings :
None.

*Sing Along (Sprite 15)
Sprite Image available : No
Yoshi Compatibility : Unknown

Description :
This sprite controls the Music House minigame, to choose prizes and the song.

Recommended Usage :
- This sprite manages a minigame, so it shouldn't be used in normal levels.
- You technically can do that, but after playing the minigame the level ends, so you can put this as a hidden secret or alternative level end.
- If you are making a Music House, putting this is NOT enough, as the Song Blocks are a different sprite which needs to be put in manually.

Settings :
- 1st/2nd/3rd/4th Prize : They explain themselves.
- ??? : Used in Newer, but doesn't do anything. You can safely ignore it.
- Song : Determines the song that will be played in the minigame.

*Bowser Bomb Dropper (Sprite 16)
Sprite Image available : No
Yoshi Compatibility : Unknown

Description :
When the specified event is triggered, a bomb falls on the specified target.

Recommended Usage :
- This can fit greatly in dark themes (underground, lava, other Bowser-related themes...), due to it being a spiked bomb (bombs in grassland is a no-no)
- Since the bomb falls from the top of the screen, make sure that place is empty, or the bomb will go through the wall and that won't look good.
- Triggers can be anything, so use this to your advantage and surprise the player with bombs.
- If the specified target doesn't exist, the game crashes, so beware.

Settings :
- Trigger Event One/Two : Both do the same thing, there's no difference between them. When this event is triggered the bomb falls, that's it.
- Target : Can be both Bowser or a player. Make sure (as i already stated) that the target exists or the game will crash.

*Song Block (Sprite 17)
Sprite Image available : No
Yoshi Compatibility : Yes

Description :
These are the actual music blocks used in Newer's Music House. They can both give out a sound or an item.

Recommended Usage :
- To make a Music House, both this and Sprite 15 are required.
- This sprite is used as a minigame prop, but it can work alone. You can use it as a music maker in a bonus area, since it only produces sound.
- This block can replace ? Blocks, since it gives items out if correctly set.
- It can also produce both sounds and items, so it's pretty flexible.

Settings :
- Item Only : Disables sound-making for the block and only gives an item
- Item : Can make this block hand an item to the player/s. Setting it to "None" disables this functionality.
- Colour/Note : To every colour corresponds a note. If you're a mean person, you can mix them up to confuse the player in the minigame.

*Bone Platform (Sprite 18)
Sprite Image available : No
Yoshi Compatibility : Yes

Description :
It's an immovable, solid-on-top Dragon Coaster piece.

Recommended Usage :
- The sprite's placement represents the right side of the platform, so make sure you position it right
- Since it's made of bones, put this sprite in dark and hot places (such as desert, lava...)
- Making a platform such as this float in mid-air looks a bit bad, so try to attach it to something (end of a rail, bridge pieces...)
- Do not set the angle to a too high value, or the character will be forced to slide off of it (as a reference, Newer does not use angles higher than 38.58°)
- Don't also set it to a vertical value, or the "wall" it creates will be solid only on one side. Just don't use this sprite as a wall, please.

Settings :
- Rotated : If not enabled, it will always be flat
- Angle : Explains itself
- Faces Right : Enable this to flip the sprite horizontally
- Model : Lets you choose what part of the Dragon Coaster to use

*SamurShai (Sprite 19)
Sprite Image available : No
Yoshi Compatibility : Probably Not

Description :
It's the boss of W4 (Sakura Village), a Shy Guy as Samurai.

Recommended Usage :
- Since this is a boss, it only makes sense to put it in a boss room.
- This boss doesn't really fit anywhere besides a Japan-themed world

Settings :
- Big Boss : Makes the sprite bigger for the second fight.

Goomba (Sprite 20)
Sprite Image available : Yes
Yoshi Compatibility : Yes

Description :
The rotten mushroom we all love to step on.

Recommended Usage :
- It can be used in almost every overworld theme, from grassland to underground, jungle, beach and even sky. In Newer, the sprite retextures allow them to fit in even more themes.
- One place where you should avoid using this enemy is where it would get in contact with water. It wouldn't make sense seeing a Goomba walking on the seabed, would it ?
- You should also avoid placing them in levels with Bowser-related themes (Tower, Castle, Airship and Lava). Why ? Because there are much better alternatives that fit in them.
- It's also recommended to not place Goombas on platforms, because they might fall before the player even reaches them. It would take out the challenge.
- If you are using a size hack, be aware that the sprite won't resize if it's in the same screen as an entrance to a zone.

Settings :
- Place Behind Other Sprites : This allows other sprites to appear in front of the Goomba. It can be useful to hide them behind other things and surprise the player. It shouldn't be misused, though.
- Load-In Animation : This affects how the Goomba reacts when it shows up (the option "Tumble" should NEVER be used. It makes the Goomba look like he's walking on the background, and at the same time it will keep rolling).
- *Scale/Size (Nyb5/7) : Allows setting a custom size for this sprite. It requires Sprite 246 (Special Event) and that sprite's function set to "Selective Sprite Hack" and to "Nybble 5/7".
- *Colour : Allows using a custom retexture. Most of the options have straightforward names, but some don't. Gloomba is dark grey, Hyper is dark blue and Bella is light pink. Santa Goombas have a santa hat on their heads. The latest versions of Reggie should have sprite images for these.

Paragoomba (Sprite 21)
Sprite Image available : Yes
Yoshi Compatibility : Yes

Description :
A winged version of the Goomba. It jumps every few seconds, and follows the player if he goes past him.

Recommended Usage :
- Being a winged enemy, the only place it fits in is the sky.
- Paragoombas are only used in World 9 in the original game for a reason : they are very hard to kill without losing your power-up.
- Since it's a difficult enemy to beat, place one of them alone, not two, not three, not more. Otherwise, the player will probably die trying to avoid them, and will get frustrated.
- If you are using a size hack, be aware that the sprite won't resize if it's in the same screen as an entrance to a zone.

Settings :
- Place Behind Other Sprites : This allows other sprites to appear in front of the Paragoomba. It can be useful to hide them behind other things and surprise the player. Since it's already hard to avoid hitting them, this shouldn't really be used.
- *Scale/Size (Nyb5/7) : Allows setting a custom size for this sprite. It requires Sprite 246 (Special Event) and that sprite's function set to "Selective Sprite Hack" and to "Nybble 5/7".
- *Colour : Allows using a custom retexture. Most of the options have straightforward names, but some don't. Gloomba is dark grey, Hyper is dark blue and Bella is light pink. Santa Goombas have a santa hat on their heads.

*Pumpkin Goomba (Sprite 22)
Sprite Image available : Yes
Yoshi Compatibility : Probably Yes

Description :
A Goomba (or Paragoomba) with a pumpkin on their head. Takes two hits to kill.

Recommended Usage :
- Being a pumpkin implies that this fits only in haunted places (Ghost House, Haunted Land...)
- The Paragoomba is a bit buggy, and hard to kill for the average player, so don't use it too often.

Settings :
- Paragoomba : Replaces the Goomba with a Paragoomba.

Horizontally/Vertically-Moving Platform (Sprite 23 and 31)
Sprite Image available : Yes
Yoshi Compatibility : Yes
Notes : These platforms are NOT controlled by lines.

Description :
Platforms that move horizontally/vertically back and forth. They can be jumped on from below, and their movement speed isn't changeable.

Recommended Usage :
- These sprites should be used in places with pits (such as sky or lava) or in vertical levels that require the player to reach a certain height (platforms help him progress). If you have a very large pit, it's recommended to use other types of platforms (such as Sprite...)
- There are two colors available for the platforms : Yellow and Grey. While the former can be used everywhere, the latter is recommended in dark enviroments only (such as lava).

Settings :
- Unknown ID : This probably does nothing. Just ignore it.
- Event Activated : This can be useful to create puzzle gimmicks, such as a switch that activates the platform and after the timer ends it disappears. It requires the next setting to be set correctly.
- Triggering Event ID : This is used to link the event activator to the platform, so that when the activator is triggered, the platform appears/disappears. It requires activating the previous setting to work.
- Slow Down : This option makes the platforms' deceleration before switching direction longer, but it normally isn't needed.
- Bowser Boss Battle Platform : Instead of the normal model, a texture from Bowser's boss bridge will be used. It's never been used, so i don't know how it looks.
- Color : Explained above. There's one duplicate for each color, but there's no difference between it and the original.
- Start Direction : Where the platforms should move first. It should be taken in mind because, when the player approaches it in-game, the platform either goes near the player and allows him to land on it, or it goes away and the player needs to wait for it to come back.
- Movement Distance : The total running distance of the platforms, a sum of left/down and right/up.
- Width : It's measured in tiles (you can activate the Grid to make accurate measurements before placement). Do not set it to 0 or 1, or it will look horrible. Even then, it's recommended to not set it lower than 4, or the player will have a hard time trying to make a perfect jump.

Buzzy Beetle (Sprite 24)
Sprite Image available : Yes
Yoshi Compatibility : Yes

Description :
An enemy similar to Koopas, but with a blue shell and immunity to fireballs. Yoshi can eat them and spit them out as shells, just as Koopas.

Recommended Usage :
- This sprite should only be used in dark themes (such as lava and underground stages). In enviroments such as Grassland, they'd stand out too much, and wouldn't fit.
- Though if you're using Newer, you can use the custom colors to make them fit in your level more.
- If you want to use the upside-down version (which falls of ceilings when the player approaches), consider making them visible and avoidable (so don't place them over the screen or in levels with low lighting, or the player won't be able to react in time.)
- If you are using a size hack, be aware that the sprite won't resize if it's in the same screen as an entrance to a zone.

Settings :
- Type : Buzzy Beetles can walk both on ground and ceiling. They can also be inserted as simple blue shells by using settings 2 and 3. Please note that "Upside-Down Shell" only works on the ground, it does not stand on ceilings.
- *Scale/Size (Nyb5/7) : Allows setting a custom size for this sprite. It requires Sprite 246 (Special Event) and that sprite's function set to "Selective Sprite Hack" and to "Nybble 5/7".
- *Colour : Allows using a custom retexture. The options are straightforward.

Spiny (Sprite 25)
Sprite Image available : Yes
Yoshi Compatibility : Yes

Description :
These quadrupedes are similar to Buzzy Beetles but, opposed to them, can only be killed with a Fire Flower. Yoshi can eat them and spit them out as shells, and can jump on them without taking damage.

Recommended Usage :
- This enemy can be used only in Desert themed levels, as their color and shape would stand out too much in other themes.
- Theoretically they could be used wherever Lakitu is present, but there are Spinies falling from the sky already, it would be too much.

Settings :
- Starts as : You can make Spinies appear as normal enemies or simple shells (the latter is never used so it's not recommended), or even look like spawned from non-existant Lakitus.
- Unknown Value 11 : Does nothing. Just ignore it.

Upside-Down Spiny (Sprite 26)
Sprite Image available : Yes
Yoshi Compatibility : Yes

Description :
The ceiling-walking version. Like the other ones, Yoshi can eat them and spit them out.

Recommended Usage :
- This enemy can be used only in Desert themed levels, as their color and shape would stand out too much in other themes.

Settings :
- Unknown Flag 8 : This option does nothing, it only creates a bug : misplaced enemies will walk in mid-air. It's recommended to keep it disabled.

Vertically/Horizontally-Moving DS Stone Block (Sprite 27 and 28)
Sprite Image available : Reggie Next only (?)
Yoshi Compatibility : Yes

Description :
The grey moving blocks from the DS game (hence the name). They are unused, and have a terrible bug. When both width and height have a value higher than 1, the inner part of the block isn't filled, and it looks horrible. Using a width or height of 1 allows to hide such behaviour.

Recommended Usage :
- They should be used in indoor environments (such as Tower, Castle and Ghost House). In most cases you will find a prettier alternative.

Settings :
- Event Activated : This can be useful to create puzzle gimmicks, such as a switch that activates the block and after the timer ends it disappears (it requires a Switch). It requires the next setting to be set correctly.
- Triggering Event ID : This is used to link the event activator to the block, so that when the activator is triggered, the block appears/disappears. It requires activating the previous setting to work.
- Slow Down : This option makes the block deceleration before switching direction longer, but it normally isn't needed.
- Height/Width : Measured in blocks
- Movement Distance : The total running distance of the platform, both left/up and right/down.

Ac_Lift_ObjBg_HMove_Big (Sprite 29) :
Doesn't load anything. It's useless.

Unused Stone Tower Block (Sprite 30)
Sprite Image available : Yes
Yoshi Compatibility : Yes

Description :
Light grey blocks that move back and forth.

Recommended Usage :
- Due to existing alternatives, this sprite is almost never used, so there's no real theme in which it fits.
- Using speed and delays, you can make them move exactly when you want them to, and create neat alternations or synchronizations.

Settings :

- Event Activated : This can be useful to create puzzle gimmicks, such as a switch that activates the block and after the timer ends it disappears (it requires a Switch). It requires the next setting to be set correctly. Please note that deactivating the block requires it to be offscreen.
- Triggering Event ID : This is used to link the event activator to the block, so that when the activator is triggered, the block appears/disappears. It requires activating the previous setting to work.
- Movement Direction : No need to explain
- Start Delay : If activated, the block will wait 2 ticks of the level's clock before moving. After the initial movement has occurred, the next two settings are the ones managing delays.
- Delay Before Going Right/Up : Sets how long it will stop after moving right/up. Each value is 2 ticks of the level's clock.
- Delay Before Going Left/Down : Sets how long it will stop after moving left/down. Each value is 2 ticks of the level's clock.
- Distance : How many tiles the block will move before returning to its starting point.
- Speed : If you set a high speed, make sure you set a delay, or the player will have a hard time passing through.
- Height/Width : In blocks

Star Coin (Sprite 32)
Sprite Image available : Yes
Yoshi Compatibility : Yes

Description
Simple collectible items, which allow to buy Hint Movies and Unlock Levels in World 9 (or unlock the entire World 9 in NewerSMBW).

Recommended Usage :
- 3 Star Coins should be in every level which doesn't fit in the following : Peach's Castle/Yoshi's House, Toad Houses/Shops, Cannons, Ambushes, Title Screen, Credits/Intro
- Make sure the Star Coins are not too easy to catch. They should involve a challenge, but it shouldn't be too hard. As you progress in the game, it's perfectly fine to make harder challenges for Star Coins, just don't go crazy and make kaizo things.

Settings :

- Target Event ID : When the Star Coin is collected, this Event ID will be activated. It can be useful to trigger wall destructions, rising lava/poison/water, enemy/platform spawning...
- Non-Timed Event : This sets whether the triggered event is timed or permanent
- Place Behind Fence : No need to explain. It can be used to make small fence puzzles and require extra effort from the player.
- Coin Number : Make sure the three coins don't have the same number, or you won't be able to collect them all. You can put more than one with the same number, if the additional ones can't be collected (see original 1-2)
- Falls when POW is Used : Other than the intended effect, it also makes the Star Coin susceptible to other objects (such as Sand Geysers, conveyor belts...)
- Place Behind Other Sprites : Can be used to hide the Star Coin, such as behind Bushes or Spike Pillars.

Event Controller - Zone Enter (Sprite 33)
Sprite Image available : No
Yoshi Compatibility : Yes

Description :
This can be used to trigger events as soon as the level starts/the zone's entered. The trigger remains in memory in the whole level, and it isn't reset by changing area or zone.

Recommended Usage :
The sprite's placement in the zone doesn't matter, but it's common sense to put it near the zone entrance. It works well with sprites that require an event ID, but you want to keep active at all times (such as Actor Spawner or Special Event).

Settings :

- Event ID : This ID will be activated/deactivated when the player enters the zone
- Time Activated : This only works if Switch Mode is set to Activate Event, and sets a duration that the Event ID will be activated for. Use 0 for unlimited time.
- Switch Mode : You can choose to activate or deactivate the specified event.
- Does Nothing/Self Destructs : Unless you are too lazy to delete the sprite yourself, you can turn this on and have the game do it for you (The sprite still remains, but it doesn't do anything.)

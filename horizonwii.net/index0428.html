 From the KB:

Actor Spawner [88]
This sprite can spawn most things that exists in the game. It can also spawn parts of sprites, for example Rocky's wrench etc.. Nybble settings are applied to all spawned sprites, but they are limited to values 0-3 instead of 0-15. See below for more information on the nybbles. Can be used together with Effect/Sound Spawner [332] to create smoke or sound effects along with the actor spawning.
Please note that sprites spawned with the Actor Spawner disappear when Mario re-enters the screen the Actor Spawner is located at. The problem can be solved by using locations and events to reactivate and therefore respawn the sprite.
Attention: Place the same sprite you want to spawn regularly into the area as well, or the game will CRASH. For example, place a Thwomp somewhere into the level in which you want to 'Actor'-spawn a Thwomp.
Spawned actors can be activated and deactivated with events. They will reappear where they left the screen.
Triggering Event ID: The actor will spawn, when this event ID is triggered. Can be switched on/off repeatedly.
Actor ID (by Name/Number/Usability): Lists all possible actors. The first number is the actor number. The second is the corresponding sprite number:
*** for actors recommended for use
--- for those with limited use or other troubles
xxx for not recommended at all
For a list of actor spawners, see → Lists → Actor Spawner List.

Timed Actor Spawner [89]
All properties are identical to the normal actor spawner above.
Exception: This sprite will spawn an actor every 2 seconds once the event is triggered. Triggering and turning the sprite off is quite tricky and I haven't totally discovered why.

Additional info:

An enemy spawned with 88 will disappear if the related event is turned off, and reappear at the same location where it disappeared once the event is turned on again.
If an enemy gets killed or falls into a pit and the event is toggled afterwards, then it will respawn at the initial position.

Possible usage scenarios with Actor Spawners:

On/Off Clouds or environmental effects
On/Off Water/Lava/Poison (or alternating)

On/Off Wind, or alternating direction
Music-synced level (Step to the Beat)
Flagpole Roulette (wait for it)
And so many more things that can be done.

On/Off Lights can be done with events (The Pit of 100 Trials new version will have just that).

The best results both for a screenshot and a video can be achieved through Dolphin, which can output at a much higher resolution than the original PAL/NTSC 480p.
In the Dolphin graphics settings, you can also increase the internal resolution and Anti Aliasing to insane numbers.
Most of those settings can be changed on-the-fly which means, you can play with shitty settings to the spot where you want to take a screenshot, freeze the game and change resolution and AA to your needs, unfreeze and take a screenie.

I need to complete this part of the KB and write a much more in-depth tutorial how to record with Dolphin (it's possible to record a playthrough with Dolphin at low resolution settings, then change the settings and let the playthrough run again to dump the frames; 4x internal resolution @ 4x AA SA at 60fps looks just amazing).

 Hello Isaac, I remember you from Youtube. Did you solve the problem by now?

If not then the error might be, that you've installed Dolphin prior to copying NewerDolphin to your drive.
If that's the case, please check -> Config -> Paths, it should read:

Default ISO: empty
DVD Root: C:\NewerDolphin
Apploader: C:\NewerDolphin\apploader.img
Wii NAND Root: C:\NewerDolphin\User\Wii

Then, make sure that
- you've deleted, backed up, or renamed "01-01.arc.LH" to "01-01.arc.LH.ORIG" in c:\NewerDolphin\Stage\
- you've placed your custom level as 01-01.arc into c:\NewerDolphin\Stage\, replacing the original level if needed
- you've started the game by selecting the "boot.dol" file through the "Open..." command (don't start any ISOs/WADs if they are loaded)

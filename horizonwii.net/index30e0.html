Over the past couple of days, I’ve started to work on a new piece of software using Python 3 and PyQt5.
This software is called Assemble Next 2018 and I hope it will help the creation of code hacks/custom sprites/et certera much more accessible.



I would like to note that Assemble Next 2018 is separate from the old Assemble Next; while that one focused on small memory hacks, I hope that 2018 will be able to be used for bigger modifications.



As I’m working on this, I’d like to hear from you - what direction would you like this to take? The original focus was for easier creation of Newer-based sprites, but depending on how everyone feels I may start to head more in the direction of ASM mods.
Additionally, what kind of interface do you think would fit? Over the last couple of days I’ve been working on a custom Python/PyQt library that makes the creation of node graphs rather easy, but I’m not sure this is the best path to take for this project and would like your advice.

You can follow my progress on Assemble Next 2018 at the GitHub repository here: https://github.com/Meorge/Assemble-Next-2018

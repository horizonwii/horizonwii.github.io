 So, i finally figured it out. It wasn't really hard, it's just that nobody has ever tried it. This tutorial will teach you how to add randomization support to whatever tileset you need. This is as easy as i could put it, i removed some of the additional code options as no feature is lost.

Requirements :

- Newer Sources (see this thread)
- Puzzle
- Ruby
- Notepad++

Pre-Steps :
- Go in NEWERSOURCES > Kamek > tools and open randtilegen.rb in Notepad++
- Open your tileset in Puzzle, to have it ready and choose the tiles

Part 1 : Adding your tileset to the script
- This will add your tileset in the randomization list. For this example, i'll use Pa1_obake_soto_u.
- Scroll until the end of the file in Notepad++, then paste the following (DON'T forget the PaX_ prefix)
g.section('TILESETNAME', 'ANOTHERTILESETNAME') do end
- Edit this template as you wish (remove the second tileset, if you don't need it)
- When filled, it will look like this :
g.section('Pa1_obake_soto_u') do end

Part 2 : Choosing tiles to randomize
- These steps will allow you to find what tiles will be randomized
- Go back to Puzzle (if it helps, toggle alpha by clicking on the related option in Tasks)



- Now you have to find the tiles' coordinates : to count the tiles, start from the top left edge, then go horizontally until the end of the line and then to the next line, and note the number when you've reached the tile in question, like in the image below (i did a mistake, there should be 80 where 78 is)


Part 3 : Adding randomization to the code
- It's now time to add the randomization code to the newly-added tileset !
- Paste the following template in a new line inbetween the start and the end of the tileset code :
g.random([TILE1, TILE2, TILE3, TILE4], :PARAMETER)
If you have many tiles that are next to each other, like in the first and second example, you can make it in a quicker way, using the following template (the first and last tiles are included) :
g.random(FIRSTTILE..LASTTILE, :PARAMETER)

The parameters work like this :
Parameters tell the game in which direction to randomize. Their usage depends on the tile's usage in Reggie :

- none is for tiles that aren't expanded, and here the game will choose a random one between the given ones
- horz is for tiles that are only expanded horizontally
- vert is for tiles that are expanded vertically
- both is for tiles that are expanded in both directions
- horz_vdouble_top and horz_vdouble_bottom are for couples of tiles, as they allow to apply randomization to both while matching each top tile (where horz_vdouble_top is applied) to the respective bottom one (where horz_vdouble_bottom is applied ).
In the spoiler below is the code for the examples :
- For example 1, we'll randomize the ground top tiles, so the code will be :
g.random(3..5, :horz)
- For example 2, we'll randomize the inside ground (therefore expanded in both directions), so this time the code will be :
g.random(19..22, :both)
- In example 3, the tiles are not next to each other, so they will have to be listed individually :
g.random([17, 33, 49, 65], :vert)

In the end, the whole code will be
g.section('Pa1_obake_soto_u') do g.random(3..5, :horz) g.random([17, 33, 49, 65], :vert) g.random(19..22, :both) end
- Repeat Part 2 and Part 3 to add other randomizations to your tileset, then save the file

Part 4 : Final Steps
- After editing the code, go back to Puzzle, and remove/shrink the objects which are going to be randomized, or randomization won't work properly
- Now open a terminal window from the script's folder, and type the following :
ruby randtilegen.rb
- This will generate a RandTiles.bin ready to be used in-game

You're done !

 This tutorial allows to add new music slots to NewerSMBW without overwriting its existing songs. Easy to edit, but hard to complete. Check here to know how to make music files.

NOTE : You can only add up to 100 songs (including the exisiting Newer ones) as the ID can't be higher than 199).

Requirements :

- Newer Sources set up and ready to be compiled (see this thread)
- Notepad++

Steps :

- Go in NEWERSOURCESFOLDER > Kamek > src and open music.cpp in Notepad++
- After scrolling down a bit, you'll find a list of the song files
- To add a new one, go to the bottom of the list, where you'll find some empty entries (if you fill all of them, create more at the very end of the list as to not make the list in Reggie mismatch)
- Add the new filename into the quotes (WITHOUT the .er extension)
- To find the new song's ID to use in Reggie, go to the top of the list and, as the first song's ID is 100, count from there
- Save, compile (refer to the thread linked in the requirements) and you're done !

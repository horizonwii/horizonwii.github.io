 Yes, you're right of course, no tiles. I was a bit too euphoric with my enumeration.

The 'Locations Property Window' allows to change the size and position, so it's not necessary either.

Very often I need to move sprites, especially coins, by a very small amount to align them nicely. Yes you can ALT-drag them, but one shaky move and you have to start over again. Let's say I need to move a coin 2px from its grid position to the right. I would end up zooming all the way in on the coin and it would probably need multiple tries until I've moved it to the desired place. If I move it too far or in any other direction, I would first need to 'snap' the coin back into the grid and try again. Sorry this is a bit hard to explain for me in English.
Sprites controlled by keyboard movement would simplify that a lot.

Another idea would be to add the 'X pos/Y pos' info to the 'Edit Selected Sprite' window (similar to locations).

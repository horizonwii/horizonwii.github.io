Posted by John Telstra
What up, ladies and gents? Today I'm gonna show you how to change the Wrist Strap screen in NSMBWii. If you wanted to change the Wrist Strap screen in NSMBWii, you bumped into the right tutorial.

Requirements

-A Windows PC

-SZS Modifier (or BrawlBox)

-The wiiStrap.arc file from NSMBWii. It's located in the /US/<Lang>US/Layout/wiiStrap (for the US/NTSC-U region, where <Lang> is the language {English/Spanish/French}),
/EU/<Lang>EU/Layout/wiiStrap (for the European/PAL region, where <Lang> is the language {English/Spanish/French/German/Italian/Dutch}), or
/JP/Layout/wiiStrap (for the Japanese/NTSC-J region) on the disc

-An image editor capable of editing PNG image files (and supports "transparency" in images, so a "solid white" background won't appear when saving the image)

Tutorial

NOTE: SZS Modifier sometimes takes a while to save the wiiStrap.arc file, so if it stops responding while saving the file, try waiting about 15 to 20 mins, then it should save. If not, use another texture editor like BrawlBox to edit the texture.

Step 1: Open the wiiStrap.arc file with SZS Modifier and open up the timg folder:


For BrawlBox:




Step 2: Now in the timg folder, you'll see a lot of TPL images.
Click on one of them, then double-click on the image preview in SZS Modifier. Then click File --> Export, and export it as a PNG image:



Click on one of them, right-click on Texture0, and export it as a PNG image:

Now get your image editor and make your edits to the PNG image with it. For example, I did a colorize, in orange, on the image (and have also done the same with the other TPL images in the wiiStrap.arc file).
Repeat Step 2 if you want to edit all, or more than one, of the TPL images.


Step 3: In SZS Modifier, go to the TPL image you want to replace. Double-click on the image preview and click File --> Import.
Choose the PNG image you just edited in Step 2:


Go to the TPL image you want to replace, right-click on Texture0, choose the PNG image you just edited in Step 2, and click Okay (don't change any of the settings/etc.):

Repeat Step 3 if you want to replace all, or more than one, of the TPL images.


Step 4: Save the wiiStrap.arc file. Then test it out on your Wii to see if it looks perfect in-game! (this is the example I did in Step 2):



NOTE: If you're using Riivolution (w/ a retail NSMBWii disc), put the line
<folder external="/nsmb/Layout"/>
in your NSMBWii XML. Replace the "nsmb" (w/o quotes) text w/ your patch folder name. Then create a folder called Layout in the patch folder (if it doesn't already exist), then put your wiiStrap.arc file in the folder.
Do not add the
disc="/Layout" create="true"/>
part in the XML line I showed above, otherwise the game won't read the wiiStrap.arc file properly when launching it with Riivolution.

This requires the wiiStrap.arc BRLYT file, a U8 archive editor (such as BrawlBox), an XML file editor (I would recommend PSPad), and Benzin the XMLYT converter (https://www.dropbox.com/s/0zdoo5sx84pylt4/benzin.zip?dl=0).

Step 1: Open the wiiStrap.arc file with BrawlBox and open up the blyt folder and export the wiiStrap.arc BRLYT file:



Step 2: Drag the wiiStrap.arc BRLYT file into the folder where Benzin is located. Open the Windows Command Prompt, switch to the folder where Benzin is located, then type this next to convert the BRLYT file to a XMLYT file:
benzin r <brlytName>.brlyt <brlytName>.xmlyt
Where <brlytName> is the name of the BRLYT file you exported with BrawlBox.

NOTE: In case if anyone gets an FAST_CWD pointer computing error with Benzin, try downloading the cygwin1.dll file here (don't get the 64-bit version, since Benzin is a 32-bit application): http://www.dlldownloader.com/cygwin1-dll/ (it's a reliable source and not malicious) and replace the existing cygwin1.dll file in the Benzin download, with the download from that site (DLLdownloader) to fix the issue.
If you've tried that out, first, and it doesn't work, try what customlevelfan78 said here: http://horizonwii.net/?page=post&id=3011


Step 3: Open the XMLYT file w/ PSPad and look for this line:
<entries name="P_back_00"> <colors> <forecolor r="255" g="255" b="255" a="0" /> <backcolor r="255" g="255" b="255" a="255" /> <colorReg3 r="255" g="255" b="255" a="255" /> <tev_k1 r="ff" g="ff" b="ff" a="ff" /> <tev_k2 r="ff" g="ff" b="ff" a="ff" /> <tev_k3 r="ff" g="ff" b="ff" a="ff" /> <tev_k4 r="ff" g="ff" b="ff" a="ff" /> </colors>
Look for this line in the XMLYT file:
<entries name="P_nintendo_00"> <colors> <forecolor r="139" g="139" b="139" a="0" /> <backcolor r="139" g="139" b="139" a="255" /> <colorReg3 r="255" g="255" b="255" a="255" /> <tev_k1 r="ff" g="ff" b="ff" a="ff" /> <tev_k2 r="ff" g="ff" b="ff" a="ff" /> <tev_k3 r="ff" g="ff" b="ff" a="ff" /> <tev_k4 r="ff" g="ff" b="ff" a="ff" /> </colors>
Step 4: Mess around w/ the RGBA Values/etc. (the numbers next to the r=, g=, b=, a= text are the RGBA Color Values) to get the color that you want. For example, I'll put the values in the r=, g=, b=, a= to zero (in the forecolor/backcolor/colorReg3 lines) to have a black color for the backdrop BG and Nintendo logo.

Step 5: Save the XMLYT file, and open the Windows Command Prompt, switch to the folder where Benzin is located, then type this next to convert the XMLYT file to a BRLYT file:
benzin m <xmlytName>.xmlyt <brlytName>.brlyt
Where <xmlytName> is the name of the XMLYT file you just modified.

Step 6: In BrawlBox, right-click on the BRLYT file and replace the BRLYT file with the one you just created (from the modified XMLYT file, with Benzin):



Step 7: Save the wiiStrap.arc file. Then test it out on your Wii to see if it looks perfect in-game! (this is the example I did to the backdrop BG and Nintendo logo in Step 4):

For just only changing the backdrop BG:

For changing both the backdrop BG and Nintendo logo:

What up, ladies and gents?


Have you ever wanted to change the save name in NewerSMBW (you know, the one in /save/SMNE/NerSMBW.sav, or /save/SMNJ/NerSMBW.sav, or /save/SMNP/NerSMBW.sav)? Well your wish has just came true!

I’ve heard someone in some forum posted for help on how to change the save name in NewerSMBW.
What he said:

Hi. Need help. In the .sav file of NewerSMBW calls NerSMBW.sav but the NSMBW original is wiimj2d.sav. How to change to NJDMBW.sav (only NJDMBW)?

Requirements

- A text string to hexadecimal converter.
- NewerSMBW's XML w/ the line: <savegame external="/save/{$__gameid}{$__region}" clone="false"/>
- An XML editor (MS Notepad. PSPad, Notepad++, etc)
- NewerSMBW
- An SD card at least 1 or 2 GB of space or a USB Stick.
- A Wii or Wii U console.

Tutorial

Step I: Download NewerSMBW at the Newer Team’s Website.


Step II: Get a text string to hexadecimal converter, I would recommend String Convertor v1.4. since it’s free and simple. Download it here. And extract the ZIP File into one folder. Now your Anti-Virus protection may not want you to download Windows applications (or .EXE files) from sites/etc. But don’t worry, this app is not a virus.


Step III: Once you’re done extracting the ZIP File, go to the folder where you extracted the program, And run the .exe file. Have these boxes in the program checked:



Usually it’ll automatically be checked, but if it doesn’t, just check them.


Step IV: Put the text you want in the ASCII String box in the program, It has to be at least 7 characters.
DO NOT PUT IN WORD THAT’S LONGER THAN 7 CHARACTERS! Otherwise the game will not accept the name and it’ll use the old save name, wiimj2d !

Also do not add the .sav extension at the end of the name! And you cannot remove the .sav extension into some other file extension. The .sav extension is mandatory - you can't remove it.

Also, you cannot put shorter text for the save name. Because putting the word “test” (which is at least 4 characters) (w/o quotes) the save name will not be test.sav when you launch the game, it’ll instead be testj2d.sav or some other random name when you see it, as it overwrites every character (in the original save file name) it uses!


After you put the word up, click the “To Hex” button in the program:



Then, the hex offset should appear in the hex box in the program. So the hex offset of the word “Example” would be hex offset, 4578616D706C65.

Now don’t close the program, leave it open. Now, get an XML editor, I would recommend PSPad since it’s free and simple, download PSPad (at the bottom of the thread). And extract it into one folder.

Step V: BACKUP YOUR NEWERSMBW XML BEFORE EDITING IT! Open the NewerSMBW XML w/ PSPad, find these lines in the XML:


<!-- Save name change -->
<!-- PAL--><memory offset="0x802F148C" value="4E6572534d42" original="7769696D6A3264"/>
<!--NTSC--><memory offset="0x802F118C" value="4E6572534d42" original="7769696D6A3264"/>
<!--JPN--><memory offset="0x802F0FAC" value="4E6572534d42" original="7769696D6A3264"/>




Copy the hex value in String Convertor v1.4. and paste in the highlighted text as it shows in the picture:



NOTE: DO NOT EDIT ANY OTHER SETTINGS IN THE MEMORY OFFSET LINES IN THE XML, OR YOU YOU MIGHT GET THIS MESSAGE HERE:



OR SOME OTHER ERROR!

After pasting the value, it should look like this:



Step VI: Hit Ctrl-S on your keyboard to save the XML.
Delete the save folder on the SD or USB Device.
BUT BE SURE TO backup the save folder on your SD or USB device before deleting it from your SD or USB device!


Step VII: Insert your SD or USB device in your Wii. Launch NewerSMBW up. And when it prompts you that the save file has been create successfully, click OK on the window:



When the screen goes black, turn off your Wii, then remove the USB or SD device from your Wii.

Step VIII: Plug in the SD or USB device in your computer. Open the root of the device on your computer, browse to /save/SMNE/ (or SMNP if you have PAL version of the game, or SMNJ if you have the Japanese version of the game), and then your save name mod change should appear here:




This tutorial/etc. is 100% Tested.


If you need help w/ any of the changing the NewerSMBW save name or have any questions about this tutorial, feel free to send me a PM or post whatever about it here. Happy Hacking.

Posted by John Telstra
Hey everyone, today I'm gonna show you how to make a multi-channel Wii RSTM {or "BRSTM"} (since nobody posted a thread about this). Now if you're willing to replace a BRSTM track in a Wii game, but it's multi-channel and doesn't fit without being multi-channel, or if unintentional behavior (e.g. weird noises) occurs in-game, you've bumped into the right tutorial.

Requirements

-A Windows PC

-Multi BRSTM Builder GUI (download here)

-An audio editor (I'd recommend Audacity, download here)

-A BRSTM player capable of playing multi-channel BRSTMs (I'd recommend BrawlBox, download here)

Tutorial

Step 1: After downloading Multi BRSTM Builder GUI, extract it to the C Drive (C:\).



Step 2: Open Audacity. You'll need a song, so drag in your audio file(s). You'll also need 2 or 4 audio files (or if you have one audio file; drag it in 2 or 4 times into Audacity).
2 audio files will make an 4 channel BRSTM which will offer 2 audio streams.
4 audio files will make an 8 channel BRSTM which will offer 4 audio streams.


For example, we'll put 4 audio files to make an 8 channel BRSTM; drag 4 audio files into Audacity, then make sure to change the input channels to 2 (Stereo):



Once you imported the files, it should look something like this:




Now click Export Multiple in Audacity, have the Numbering after File name Prefix and
Tracks options checked. Now choose the Export Location, make sure you set the location to a place on your PC that you're going to remember.

Set the export format as Microsoft WAV Signed 16bit PCM (or Other uncompressed files,
and choosing Options..., and choosing the header as WAV [Microsoft] and Encoding as Signed 16bit PCM). And then click the Export Button in the window, then hit "OK" to all files that are going to be exported:







Step 3: Reopen Audacity and search for your .wav audio files that you exported:



And drag the files into Audacity. After that, change the input channel to 1 (Mono):



And split ALL the audio tracks to mono:



When you're done, it should look something like this:



Then play it. And it may sound a bit scratchy/loud, but keep it like that since it won't turn scratchy/loud when you render it for whatever reason (I don't know why nor anyone else does).


Click Export Multiple again in Audacity. Make sure the Export Location is the same directory you exported the other songs in Step 2.

Set the export format as Microsoft WAV Signed 16bit PCM (or Other uncompressed files,
and choosing Options..., and choosing the header as WAV [Microsoft] and Encoding as Signed 16bit PCM). And then click the Export Button in the window, then hit "OK" to all files that are going to be exported:





Step 4: Now, there may be 12 audio files. Delete ALL the stereo-track files:



And rename the file (for channel 1) to "1", and
the other file (for channel 2) to "2", and
the other file (for channel 3) to "3" and
the other file (for channel 4) to "4", etc:




Drag all the audio files in the wav folder in the Multi BRSTM Builder GUI folder:





Step 5: Open BrawlBox, go to New --> BRSTM Audio Stream, then locate your .wav file. And loop it to whatever point(s) you like (that seems to fit you),
then write down the Loop End Samples and Loop Start Samples number (click the play button to preview the loop, but check the loop option first). Then exit out of BrawlBox (don't save/convert the WAV):









Step 6: Open Multi BRSTM Builder GUI and enter in the Loop Start Samples and Loop End Samples you've written down, Make sure you check the loop option!
Then select how many audio channels you want for the BRSTM. Since I'm using 8 channels, I'll choose 8 channels. And for the BRSTM Output Filename, name it whatever you want, but do not add the extension (.brstm) to the end of the file name.

It should look something like this:




Once you're done, click Build Batch File and choose exit. Now click on the build.bat batch file and wait. You'll see it's creating files like 1.txt, 2.txt, etc.
When the batch file creates those .txt files, it's encoding the BRSTM. After it's done, open the out folder in the Multi BRSTM Builder GUI folder, and you should see your multi-channel BRSTM:





Step 7: Open the multi-channel BRSTM with BrawlBox and check the Loop option. Play it and listen to all the streams to see how it sounds when you looped it/etc.
Then test it on your Wii. I hope this works for you. If you all need help, PM me or post in my topic right here. Happy Hacking.

NOTE: If your BRSTMs are stopping after a certain amount of time (e.g. 30 or 50 secs) when testing it in-game, you'll need to patch the BRSAR file to get it to play properly and don't stop. For instructions on how to patch the BRSAR file, see: http://horizonwii.net/?page=thread&id=124. And for a list of sound files in NSMBWii, see this thread: http://horizonwii.net/?page=thread&id=87.

Posted by John Telstra
What up, ladies and gents? Today I'm gonna show you how to change the Wrist Strap screen in NSMBWii. If you wanted to change the Wrist Strap screen in NSMBWii, you bumped into the right tutorial.

Requirements

-A Windows PC

-SZS Modifier (or BrawlBox)

-The wiiStrap.arc file from NSMBWii. It's located in the /US/<Lang>US/Layout/wiiStrap (for the US/NTSC-U region, where <Lang> is the language {English/Spanish/French}),
/EU/<Lang>EU/Layout/wiiStrap (for the European/PAL region, where <Lang> is the language {English/Spanish/French/German/Italian/Dutch}), or
/JP/Layout/wiiStrap (for the Japanese/NTSC-J region) on the disc

-An image editor capable of editing PNG image files (and supports "transparency" in images, so a "solid white" background won't appear when saving the image)

Tutorial

NOTE: SZS Modifier sometimes takes a while to save the wiiStrap.arc file, so if it stops responding while saving the file, try waiting about 15 to 20 mins, then it should save. If not, use another texture editor like BrawlBox to edit the texture.

Step 1: Open the wiiStrap.arc file with SZS Modifier and open up the timg folder:


For BrawlBox:




Step 2: Now in the timg folder, you'll see a lot of TPL images.
Click on one of them, then double-click on the image preview in SZS Modifier. Then click File --> Export, and export it as a PNG image:



Click on one of them, right-click on Texture0, and export it as a PNG image:

Now get your image editor and make your edits to the PNG image with it. For example, I did a colorize, in orange, on the image (and have also done the same with the other TPL images in the wiiStrap.arc file).
Repeat Step 2 if you want to edit all, or more than one, of the TPL images.


Step 3: In SZS Modifier, go to the TPL image you want to replace. Double-click on the image preview and click File --> Import.
Choose the PNG image you just edited in Step 2:


Go to the TPL image you want to replace, right-click on Texture0, choose the PNG image you just edited in Step 2, and click Okay (don't change any of the settings/etc.):

Repeat Step 3 if you want to replace all, or more than one, of the TPL images.


Step 4: Save the wiiStrap.arc file. Then test it out on your Wii to see if it looks perfect in-game! (this is the example I did in Step 2):



NOTE: If you're using Riivolution (w/ a retail NSMBWii disc), put the line
<folder external="/nsmb/Layout"/>
in your NSMBWii XML. Replace the "nsmb" (w/o quotes) text w/ your patch folder name. Then create a folder called Layout in the patch folder (if it doesn't already exist), then put your wiiStrap.arc file in the folder.
Do not add the
disc="/Layout" create="true"/>
part in the XML line I showed above, otherwise the game won't read the wiiStrap.arc file properly when launching it with Riivolution.

This requires the wiiStrap.arc BRLYT file, a U8 archive editor (such as BrawlBox), an XML file editor (I would recommend PSPad), and Benzin the XMLYT converter (https://www.dropbox.com/s/0zdoo5sx84pylt4/benzin.zip?dl=0).

Step 1: Open the wiiStrap.arc file with BrawlBox and open up the blyt folder and export the wiiStrap.arc BRLYT file:



Step 2: Drag the wiiStrap.arc BRLYT file into the folder where Benzin is located. Open the Windows Command Prompt, switch to the folder where Benzin is located, then type this next to convert the BRLYT file to a XMLYT file:
benzin r <brlytName>.brlyt <brlytName>.xmlyt
Where <brlytName> is the name of the BRLYT file you exported with BrawlBox.

NOTE: In case if anyone gets an FAST_CWD pointer computing error with Benzin, try downloading the cygwin1.dll file here (don't get the 64-bit version, since Benzin is a 32-bit application): http://www.dlldownloader.com/cygwin1-dll/ (it's a reliable source and not malicious) and replace the existing cygwin1.dll file in the Benzin download, with the download from that site (DLLdownloader) to fix the issue.
If you've tried that out, first, and it doesn't work, try what customlevelfan78 said here: http://horizonwii.net/?page=post&id=3011


Step 3: Open the XMLYT file w/ PSPad and look for this line:
<entries name="P_back_00"> <colors> <forecolor r="255" g="255" b="255" a="0" /> <backcolor r="255" g="255" b="255" a="255" /> <colorReg3 r="255" g="255" b="255" a="255" /> <tev_k1 r="ff" g="ff" b="ff" a="ff" /> <tev_k2 r="ff" g="ff" b="ff" a="ff" /> <tev_k3 r="ff" g="ff" b="ff" a="ff" /> <tev_k4 r="ff" g="ff" b="ff" a="ff" /> </colors>
Look for this line in the XMLYT file:
<entries name="P_nintendo_00"> <colors> <forecolor r="139" g="139" b="139" a="0" /> <backcolor r="139" g="139" b="139" a="255" /> <colorReg3 r="255" g="255" b="255" a="255" /> <tev_k1 r="ff" g="ff" b="ff" a="ff" /> <tev_k2 r="ff" g="ff" b="ff" a="ff" /> <tev_k3 r="ff" g="ff" b="ff" a="ff" /> <tev_k4 r="ff" g="ff" b="ff" a="ff" /> </colors>
Step 4: Mess around w/ the RGBA Values/etc. (the numbers next to the r=, g=, b=, a= text are the RGBA Color Values) to get the color that you want. For example, I'll put the values in the r=, g=, b=, a= to zero (in the forecolor/backcolor/colorReg3 lines) to have a black color for the backdrop BG and Nintendo logo.

Step 5: Save the XMLYT file, and open the Windows Command Prompt, switch to the folder where Benzin is located, then type this next to convert the XMLYT file to a BRLYT file:
benzin m <xmlytName>.xmlyt <brlytName>.brlyt
Where <xmlytName> is the name of the XMLYT file you just modified.

Step 6: In BrawlBox, right-click on the BRLYT file and replace the BRLYT file with the one you just created (from the modified XMLYT file, with Benzin):



Step 7: Save the wiiStrap.arc file. Then test it out on your Wii to see if it looks perfect in-game! (this is the example I did to the backdrop BG and Nintendo logo in Step 4):

For changing both the backdrop BG and Nintendo logo:

 If you make custom graphics and sprites, then the best order is:

1. ASM hacks (custom sprites etc.)
2. Custom graphics (tilesets, backgrounds)
3. Levels
(4. World maps - can be ignored)
5. Logo
6. Patch

This order should help you avoid situations when e.g. Your ASM isn't ready so you can't make a level. If you don't make custom code, ignore point 1. Point 4 is most useful in Newer engine, because there is map editor, but actual NSMBW maps can be edited too, but it's harder then.

If you're using custom tilesets in NSMBW with Riivolution, you need to replace the line in your XML:
<Wiidisc version="1">
with
<wiidisc version="1" shiftfiles="true">
or the game will crash, regardless whether the tileset exists or not.

You'll need to use Koopatlas in order to edit the NewerSMBW Maps.
Here's a guide on how to use it: http://horizonwii.net/?page=thread&id=127

In addition, be aware that the Map files (.kpbin) and Map Tileset files (.bin) in the NewerSMBW mod, will not open in Koopatlas (since Koopatlas only reads .kpmap files for the Maps, and .arc files for the Map Tilesets).

So, you'll have to download the .kpmap Map files here, and .arc Map Tileset files here, in order to use the NewerSMBW Maps and Map Tilesets with Koopatlas.

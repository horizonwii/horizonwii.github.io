Sorry if I didn't reply to you guys earlier, but is the user you're on (in the Windows 10 WSL subsystem), is the ROOT user (which is UID 0)?

To check, type this command in the WSL subsystem:
echo $UID
to see if it returns 0.


If it returns 0, the only thing you can do is, uninstall the WSL subsystem and install it again and install the NewerSMBW sources again on the WSL subsystem to see if it works.

Otherwise, I don't know the answer to those problems you are both having, because I never had issues like that before, and the compiling works for me perfectly on the WSL subsystem on my Windows 10 computer.
